# 战火数据分析工具

这是一个用于分析战火游戏玩家数据的Python工具，支持公会成员数据采集、战斗数据分析和Excel报告生成。

## 功能特点

### 1. 数据采集
- 自动获取指定公会的成员数据
- 支持多日期范围的数据收集
- 异步并发请求，提高采集效率
- 内置重试机制，确保数据完整性

### 2. 数据分析
- 战力分组分析
  * 按战力区间自动分组（1.5亿-5亿+）
  * 每个战力段的击杀、死亡阈值自适应
  * 战力增长趋势分析

- 活跃度分析
  * 多维度活跃度计算
  * 战斗参与度评估
  * 战力增长评估
  * 分级活跃度统计（高、中、低活跃）

- 战斗数据分析
  * 击杀/死亡数据统计
  * 击杀效率分析
  * 战斗表现评估

### 3. 数据可视化
- 生成Excel报告，包含多个数据表：
  * 用户总览
  * 每日统计
  * 战力分组统计
  * 多维度图表展示

## 使用方法

### 1. 数据采集

```bash
python main.py --gid <公会ID> --current-date <当前日期> --start-date <开始日期> --end-date <结束日期>
```

参数说明：
- `--gid`: 公会ID（必需）
- `--current-date`: 当前日期，格式YYYYMMDD（可选，默认今天）
- `--start-date`: 统计开始日期，格式YYYYMMDD（可选，默认今天）
- `--end-date`: 统计结束日期，格式YYYYMMDD（可选，默认今天）
- `--output-dir`: 输出目录（可选，默认"warpath_data"）
- `--max-concurrent`: 最大并发请求数（可选，默认10）
- `--max-retries`: 最大重试次数（可选，默认3）
- `--retry-delay`: 重试延迟秒数（可选，默认2）

### 2. 数据处理

```bash
python Neibu.py <JSON文件路径> <开始日期> <结束日期> <输出Excel文件路径>
```

参数说明：
- JSON文件路径：数据采集阶段生成的JSON文件
- 开始日期：统计开始日期（YYYY-MM-DD）
- 结束日期：统计结束日期（YYYY-MM-DD）
- 输出Excel文件路径：生成的Excel报告路径

  * 平均战斗活跃度

- **每日统计表**

  * 每日死亡数据

  * 每日活跃度评分

- **战力分组表**

  * 组内成员表现
  * 组间对比分析

### 2. 数据可视化图表

- 用户击杀数分布图
- 每日击杀与被击杀趋势图
- 用户击杀比分布图
- 战力与击杀效率关系图
- 用户活跃度分析图
- 用户战力分布图

## 注意事项

1. 数据采集需要稳定的网络连接
2. 大量数据处理可能需要较长时间
3. Excel文件可能较大，请确保足够的磁盘空间
4. 建议定期备份原始数据
