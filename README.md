# 战火公会数据分析工具

这是一个用于分析战火游戏公会成员数据的工具，提供了图形用户界面，可以方便地获取、处理和分析公会成员的战斗数据，并生成详细的Excel报告。支持单个公会分析和多公会批量分析。

## 功能特性

### 图形用户界面
- 现代化的界面设计，支持屏幕分辨率自适应
- 直观的日期选择器，支持日历选择
- 实时日志显示和进度反馈
- 支持单公会和多公会分析模式切换
- 可配置的并发请求和重试参数

### 数据获取
- 支持异步并发请求，提高数据获取效率
- 内置重试机制，自动处理网络异常
- 支持自定义并发数和重试参数
- 自动保存原始数据，方便后续分析
- 支持多公会批量数据获取

### 数据分析
- 按战力分组分析成员表现
- 计算每日战斗统计
- 分析活跃度指标
- 生成多维度数据报告
- 支持多公会数据对比分析

### 数据导出
- 生成结构化的Excel报告
- 包含多个数据维度的工作表
- 自动生成数据可视化图表
- 支持自定义输出格式
- 支持多公会数据汇总报告

## 使用方法

### 图形界面启动
```bash
python gui_qt.py
```

### 命令行启动
```bash
python main.py --gid <公会ID> --current-date <日期> --start-date <开始日期> --end-date <结束日期> --output-dir <输出目录>
```

### 多公会分析
```bash
python main.py --gids <公会ID列表> --current-date <日期> --start-date <开始日期> --end-date <结束日期> --output-dir <输出目录>
```

示例：
```bash
python main.py --gids 1001,1002,1003 --current-date 20240330 --start-date 20240301 --end-date 20240330 --output-dir warpath_data
```

### 参数说明

- `--gid`: 单个公会ID（单公会分析时使用）
- `--gids`: 多个公会ID，用逗号分隔（多公会分析时使用）
- `--current-date`: 当前日期，格式为YYYYMMDD
- `--start-date`: 开始日期，格式为YYYYMMDD
- `--end-date`: 结束日期，格式为YYYYMMDD
- `--output-dir`: 输出目录（可选，默认为'warpath_data'）
- `--max-concurrent`: 最大并发请求数（可选，默认为10）
- `--max-retries`: 最大重试次数（可选，默认为3）
- `--retry-delay`: 重试延迟（秒）（可选，默认为2）
- `--compare`: 是否生成公会间对比报告（可选，默认为True）

## 图形界面使用说明

### 基本操作
1. 启动程序后，在"输入参数"区域填写必要信息：
   - 选择分析模式：单公会或多公会
   - 使用日历选择器选择日期范围
   - 设置输出目录
   - 配置并发和重试参数

2. 点击"开始分析"按钮开始处理数据

3. 在"运行日志"区域查看实时进度

4. 分析完成后，可在输出目录查看生成的报告

### 注意事项
- 单公会和多公会模式不能同时使用
- 多公会分析需要至少输入两个公会ID
- 开始日期不能晚于结束日期
- 建议根据网络情况调整并发数和重试参数

## 活跃度计算

活跃度基于以下维度综合计算：
1. 击杀数（达到阈值30%）
2. 死亡数（达到阈值30%）
3. 战力增长（达到阈值50%）
4. 战斗活跃度（达到30分）

战斗活跃度得分（0-100分）计算公式：
```
battle_score = min(100, (daily_kills / 200 + daily_deaths / 300) * 50)
```

## 注意事项

1. 确保网络连接稳定
2. 建议使用代理或VPN以避免请求限制
3. 大量数据获取时注意控制并发数
4. 定期备份原始数据
5. 多公会分析时注意控制并发请求数，避免触发服务器限制
